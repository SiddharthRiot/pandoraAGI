# Contributing to PandoraAGI

PandoraAGI is not a side project. It is an attempt to change who controls the most powerful technology ever built.

Every contribution matters.

---

## Before You Start

Understand what you are contributing to.

AGI development is not a hackathon. This is long-term, serious work. Read the README fully. Understand the vision. If you are here just to add a line to your resume this is not the right place.

If you genuinely believe AGI should belong to humanity welcome.

---

## How to Contribute

### 1. Find an area

PandoraAGI has five core layers. Pick what matches your skills.

| Layer | What it needs | Skills |
|---|---|---|
| The Living Model | Continual learning, model architecture, training pipelines | Python, PyTorch, ML |
| The Substrate | P2P networking, distributed compute, fault tolerance | Distributed systems, networking |
| The Sensory Layer | Data ingestion, real-time pipelines, multimodal processing | Data engineering, Python |
| The Governance Layer | On-chain voting, audit trails, protocol design | Blockchain, cryptography |
| The Interface | 3D visualization, real-time rendering, UX | Three.js, WebGL, React |

### 2. Check open issues

Look for issues tagged:
- `good-first-issue` - start here if you are new
- `research` - theoretical work, no code required
- `core` - foundational architecture work
- `help-wanted` - active priority

### 3. Fork and branch

```bash
git clone https://github.com/yourusername/pandoraAGI
cd pandoraAGI
git checkout -b your-feature-name
```

### 4. Make your changes

Keep it focused. One feature or fix per pull request.

### 5. Submit a pull request

- Clear title describing what you did
- Why you did it
- Any tradeoffs or decisions made

---

## Contribution Types

You do not have to write code to contribute.

- **Research** - literature reviews, architecture proposals, alignment analysis
- **Documentation** - technical writing, tutorials, explanations
- **Code** - any of the five layers
- **Testing** - benchmarks, stress tests, edge cases
- **Design** - system design, protocol design, interface design
- **Translation** - make PandoraAGI accessible globally

---

## Code Standards

- Python: follow PEP 8
- JavaScript/TypeScript: ESLint + Prettier
- All code must be documented
- All new features must include tests
- No external dependencies without discussion first

---

## Principles

**Openness** - Everything is public. No private forks, no closed discussions.

**No ego** - The best idea wins, not the loudest voice.

**Long-term thinking** - We are not shipping a product. We are building infrastructure for the future of intelligence.

**Alignment first** - If a contribution increases capability without addressing safety, it will not be merged.

---

## Communication

- Open an issue before starting large work avoid duplicate effort
- Be direct, be respectful, be constructive
- Discussions are public everything is on record

---

## What We Do Not Accept

- Contributions designed to centralize control
- Code that introduces backdoors or surveillance
- Work that benefits a single corporation or government
- Anything that violates the Apache 2.0 license

---

## First Contribution Ideas

Not sure where to start?

- Read the architecture docs and raise questions
- Write a research summary on distributed training approaches
- Set up the local dev environment and document any issues
- Propose improvements to this file
